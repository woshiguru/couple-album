# 更新接口使用说明

## 这个功能是干什么的？

程序内置了一个"离线更新"功能。你不需要重新安装程序，只需要把更新包放到指定文件夹，程序就能自动升级。

---

## 怎么用？（3 步完成更新）

### 第一步：拿到更新包

更新包是一个 `.zip` 压缩文件，文件名格式为：

```
update_版本号.zip
```

比如：`update_1.2.0.zip`

### 第二步：放到更新文件夹

把更新包复制到程序目录下的这个位置：

```
storage/updates/
```

> 完整的文件夹路径：`你的程序目录/storage/updates/`

### 第三步：在程序里点"检查更新"

1. 打开程序
2. 点击右上角 **🔧 管理**
3. 在右侧管理面板底部找到 **📦 程序更新**
4. 点击 **检查更新**
5. 如果有新版本，会弹窗提示，点击 **立即更新**
6. 程序自动替换文件并重启，更新完成 ✅

---

## 更新包里面有什么？

一个合格的更新包（zip 文件）里面必须有这些东西：

```
update_1.2.0.zip
│
├── update.json          ← 必须有！更新说明文件
├── index.html           ← 要替换的前端页面（可选）
└── src/
    └── app.py           ← 要替换的后端代码（可选）
```

### update.json 怎么写？

`update.json` 是更新包的"说明书"，告诉程序这个更新包是做什么的、要替换哪些文件。

格式如下：

```json
{
  "version": "1.2.0",
  "description": "这次更新修了什么、加了什么",
  "files": [
    { "path": "index.html", "action": "replace" },
    { "path": "src/app.py", "action": "replace" }
  ]
}
```

各字段说明：

| 字段 | 作用 | 示例 |
|------|------|------|
| `version` | 新版本号，必须比当前版本高，否则程序不会识别 | `"1.2.0"` |
| `description` | 更新内容说明，更新后会展示给用户看 | `"修复了收藏功能的bug"` |
| `files` | 要替换的文件列表 | 见下方 |
| `files[].path` | 文件路径（相对于程序根目录） | `"index.html"` 或 `"src/app.py"` |
| `files[].action` | 操作类型，目前只支持 `replace`（替换） | `"replace"` |

---

## 版本号规则

版本号格式为 `主版本.次版本.修订号`，例如 `1.2.0`。

程序会自动比较版本号大小：

| 当前版本 | 更新包版本 | 结果 |
|---------|-----------|------|
| 1.0.0 | 1.1.0 | ✅ 可以更新 |
| 1.0.0 | 1.0.1 | ✅ 可以更新 |
| 1.1.0 | 1.1.0 | ❌ 版本相同，不会更新 |
| 1.2.0 | 1.1.0 | ❌ 更新包版本更低，不会更新 |

---

## 更新过程会发生什么？

```
点击"检查更新"
      ↓
程序扫描 storage/updates/ 里的所有 .zip 文件
      ↓
读取每个 zip 里的 update.json，找到版本号最高的
      ↓
对比当前版本，如果有新版本 → 弹窗提示
      ↓
用户点击"立即更新"
      ↓
程序自动做以下事情：
  1. 备份当前文件到 storage/updates/backup_日期/
  2. 解压更新包
  3. 按 update.json 的 files 列表替换文件
  4. 自动更新 metadata.json 里的版本号
  5. 重启程序
      ↓
重启后就是新版本了 ✅
```

---

## 更新失败了怎么办？

每次更新前，程序会自动备份当前文件。备份存放在：

```
storage/updates/backup_年月日_时分秒/
```

如果更新出问题，可以手动从备份文件夹恢复文件。

---

## 常见问题

**Q：更新包可以只更新 index.html 吗？**
A：可以。`files` 列表里只写 `index.html` 就行，不用包含所有文件。

**Q：更新包可以只更新 app.py 吗？**
A：可以。同理，只写 `src/app.py`。

**Q：可以同时更新多个文件吗？**
A：可以。在 `files` 列表里添加多个条目即可。

**Q：版本号写错了会怎样？**
A：如果更新包的版本号 ≤ 当前版本，程序会提示"当前已是最新版本"，不会执行更新。

**Q：更新后程序没有变化？**
A：检查 update.json 里的 `files` 路径是否正确，文件是否真的被替换了。也可以查看 `storage/updates/backup_*/` 里的备份文件对比。

## 示例：制作一个完整的更新包

假设当前版本是 1.1.0，你要发布 1.2.0 版本，修复了收藏功能。

**1. 准备更新内容**
- 修改好的 `index.html`
- 修改好的 `src/app.py`

**2. 写 update.json**
```json
{
  "version": "1.2.0",
  "description": "修复了收藏功能重启后丢失的问题",
  "files": [
    { "path": "index.html", "action": "replace" },
    { "path": "src/app.py", "action": "replace" }
  ]
}
```

**3. 打包成 zip**

把上面 3 个文件打包成 `update_1.2.0.zip`：
```
update_1.2.0.zip
├── update.json
├── index.html
└── src/
    └── app.py
```

**4. 放到更新文件夹**

把 `update_1.2.0.zip` 复制到 `storage/updates/` 目录。

**5. 在程序里检查更新**

打开程序 → 管理 → 检查更新 → 立即更新 → 完成！
